# 中文发布说明

生成发布说明时遵循以下流程：

1. 使用 `git status --short` 确认变更范围。
2. 使用 `git diff --stat` 和 `git diff` 阅读实际变更，不猜测未展示的行为。
3. 按“新增、修复、改进、兼容性”组织内容；没有内容的章节省略。
4. 描述用户可感知的结果，避免逐文件复述实现细节。
5. 最后列出已经明确执行过的验证；没有验证时写“尚未验证”。

除非用户明确要求，否则不要修改文件、创建提交或发布版本。
