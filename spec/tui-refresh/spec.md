# Windcode TUI 启动页与命令菜单 Spec

## 背景

当前终端界面启动后直接展示紧凑标题栏和空白聊天区，缺少清晰的品牌欢迎态；用户在输入框键入 `/` 时也没有任何可见反馈，只有提交完整命令后才会解析。这使首次使用缺乏引导，斜杠命令不易发现，并且界面层级与成熟编码 Agent 的终端体验存在差距。

本次改造借鉴 Codex、Claude Code、Pi Agent 和 OpenCode 的信息组织思路，但保持 Windcode 自有品牌、中文文案及现有功能边界。

## 目标

- 为新会话提供以 `windcode` 为核心品牌信号的启动欢迎态。
- 让斜杠命令在输入过程中可发现、可筛选、可通过键盘完成选择。
- 首次提交后自然切换为适合长时间工作的紧凑对话布局。
- 统一启动态、对话态、命令菜单、状态信息和消息流的视觉层级。
- 保持现有会话、模型、权限、审批、工具调用和流式输出行为不变。

## 功能需求

- F1：新会话启动时显示居中的 `windcode` 品牌标识、任务输入区域、当前模型、权限模式和沙箱状态。
- F2：启动欢迎态必须直接提供可输入、可提交的主输入框，不增加独立落地页或额外确认步骤。
- F3：用户提交第一条普通消息后，欢迎态自动切换为紧凑对话态；后续对话期间不再占用大面积空间显示品牌标识。
- F4：恢复已有会话时直接进入紧凑对话态，不显示新会话欢迎态。
- F5：用户在输入框首字符键入 `/` 时立即显示命令菜单，不要求先按 Enter 或其他快捷键。
- F6：命令菜单展示当前可用命令及中文说明，并随用户继续输入实时筛选；无匹配项时隐藏菜单。
- F7：命令菜单支持方向键移动选择、Enter 执行选中命令、Tab 补全选中命令、Esc 关闭菜单，关闭后焦点仍在输入框。
- F8：命令菜单可见时，Enter 优先执行菜单选中项；菜单不可见时，Enter 保持现有消息或完整命令提交行为。
- F9：提交命令、普通消息、创建新会话或恢复会话后，命令菜单不得残留或遮挡输入区。
- F10：执行新建会话后重新显示欢迎态；选择恢复会话后保持紧凑对话态。
- F11：启动态和对话态均使用中文界面文案，品牌名称统一显示为 `windcode`，不出现参考产品名称。
- F12：紧凑对话态应清晰区分用户消息、助手回复、推理完成状态、工具调用和底部运行状态，同时延续现有流式更新方式。
- F13：窄终端和常见桌面终端尺寸下，品牌标识、输入框、命令菜单、消息流和状态信息不得重叠或被截断到无法使用。

## 非功能需求

- N1：所有交互必须可仅使用键盘完成，输入焦点切换可预测。
- N2：输入 `/` 到命令菜单出现之间不得有可感知延迟，菜单更新不得阻塞模型流式输出。
- N3：视觉风格保持克制、低干扰、高信息密度，避免装饰性卡片堆叠和过多色彩。
- N4：颜色和选中状态在深色终端背景下具有足够对比度，不依赖颜色作为唯一状态提示。
- N5：自动化测试不依赖真实模型、外部网络或特定终端背景图片。
- N6：现有公共开发接口、会话数据格式和模型事件格式保持兼容。

## 不做的事

- 不复制 Codex、Claude Code、Pi Agent 或 OpenCode 的源码、Logo 或产品文案。
- 不引入背景图片、透明终端效果或依赖终端模拟器的视觉特性。
- 不新增 Agent 切换、Provider 连接、会话导出、Diff 查看器或编辑器集成等命令。
- 不实现鼠标优先的命令面板；鼠标支持不是本轮验收条件。
- 不改变模型调用、上下文管理、工具执行、权限策略或会话持久化语义。
- 不在启动页增加教程、功能介绍长文或营销内容。

## 验收标准

- AC1（F1、F2、F11）：启动新会话时，首屏可观察到居中的 `windcode` 品牌、可聚焦输入框以及模型、权限和沙箱状态，所有辅助文案为中文。
- AC2（F3、F12）：提交第一条普通消息后，大型欢迎态消失并切换为紧凑对话布局，用户消息和流式助手回复正常显示。
- AC3（F4、F10）：恢复已有会话时不显示大型欢迎态；执行新建会话后欢迎态重新出现。
- AC4（F5、F6）：输入 `/` 后立即出现完整命令菜单，继续输入 `/mo` 时只保留匹配命令，输入无匹配前缀时菜单隐藏。
- AC5（F7、F8）：菜单可见时，方向键可改变选中项，Tab 可补全，Enter 可执行选中命令，Esc 可关闭菜单且输入框保持焦点。
- AC6（F9）：执行命令、提交消息、新建或恢复会话后，命令菜单均关闭且不遮挡聊天内容。
- AC7（F12）：现有文本流、推理状态、工具块、审批、提问、错误和完成状态在改造后的紧凑布局中仍能正常显示和交互。
- AC8（F13、N1、N4）：在 40×24、80×24 和 120×36 终端尺寸下，关键内容无重叠，最长命令说明可阅读，全部核心操作可通过键盘完成。
- AC9（N2、N5、N6）：格式检查、严格类型检查、TUI 集成测试和全量测试通过，测试过程中不访问真实模型或网络，现有公共接口保持兼容。
