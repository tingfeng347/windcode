# Windcode 扩展运行时 Checklist

> 每一项必须通过自动化测试、进程/连接探针或用户可见行为验证。实现文件存在不等于
> 验收通过；先记录实际证据，再把 `[ ]` 改为 `[x]`。

## 能力目录、作用域与快照

- [ ] **AC1：统一能力目录**包含能力名称、类型、稳定 ID、来源、作用域、启用状态、信任状态、权限需求和诊断状态。（验证：配置四级来源后分别通过 CLI、TUI、SDK 列出并比较字段）
- [ ] **AC1：作用域优先级**固定为内建、用户、项目、运行级，高优先级独立资源覆盖低优先级资源并显示 `shadowed_by` 关系。（验证：四级定义同名 Skill、Hook 和 MCP Server，检查胜出项及覆盖链）
- [ ] **AC1：插件命名空间**保证插件组件不会被独立资源或其他插件静默覆盖。（验证：构造重复插件 ID 和组件 ID，期望明确冲突诊断）
- [ ] **AC1、AC24：确定性结果**不随文件系统遍历顺序或异步完成顺序变化。（验证：随机化发现顺序并重复运行，比较能力 ID、排序、覆盖和诊断）
- [ ] **AC2：未信任项目资源只可检查**，Skills、Hooks、MCP 和插件不得激活、执行或注入模型上下文。（验证：在未信任临时工作区加载四类项目资源并检查执行与模型请求计数为零）
- [ ] **AC2：信任后显式生效**；建立工作区信任但未重载时行为不变，显式重载后新运行才能使用项目资源。（验证：依次检查信任前、信任后未重载、重载后状态）
- [ ] **AC6：运行持有不可变快照**；运行开始后禁用插件并重载，原运行仍使用原 generation，新运行使用新 generation。（验证：两个跨重载并发运行分别调用插件能力）
- [ ] **AC6：失败重载不破坏旧快照**；必需扩展错误阻止候选发布，已有运行和之后的查询仍能读取旧快照。（验证：引入必需组件错误并比较 reload 前后 generation）

## 插件清单与本地安装

- [ ] **AC3：组合清单**可以声明插件元数据、兼容版本、Skills、MCP Servers、Hooks、命令、权限和数据需求。（验证：解析完整测试插件并检查所有组件的插件命名空间）
- [ ] **AC3：严格清单校验**拒绝非法 ID、重复 ID、未知必需字段和不兼容 Windcode 版本。（验证：逐类构造无效清单并检查阶段、来源、类别和修复建议）
- [ ] **AC3、AC23：插件路径安全**拒绝绝对路径、`..`、符号链接越界、设备文件、FIFO 和跨插件路径。（验证：恶意清单与安装目录路径测试）
- [ ] **AC4：本地安装记录来源与摘要**，安装内容使用稳定 SHA-256，重复安装相同摘要保持幂等。（验证：安装两次同一目录并检查安装记录和目标目录）
- [ ] **AC4：同版本不同内容不被静默替换**，系统返回来源明确的内容冲突诊断。（验证：修改文件但保留版本号后再次安装）
- [ ] **AC4：安装阶段零执行**；检查和安装不得启动 MCP、执行 Hook/脚本、导入宿主代码或展开 Skill 正文。（验证：使用带执行哨兵的插件并检查进程、网络与文件访问记录）
- [ ] **AC4：安装原子性**；复制、摘要复核或状态写入失败时不发布半成品安装。（验证：在各发布阶段注入失败并检查目标目录与状态文件）
- [ ] **AC5：声明式命令路由**可以触发对应 Skill、MCP Prompt 或已有能力，并保留插件来源。（验证：分别调用三种合法目标）
- [ ] **AC5：命令不能执行任意宿主代码**；带 Shell、模块、入口点或任意代码字段的清单被拒绝。（验证：构造恶意命令条目并确认解析失败且无副作用）
- [ ] **AC18：插件数据隔离**；两个插件只能访问各自宿主管理的数据目录，不能声明任意持久路径或跨插件访问。（验证：两个插件互相读取和路径逃逸测试）
- [ ] **AC18：插件状态文件原子且私有**，损坏状态返回诊断而不静默重置用户数据。（验证：检查文件权限并注入截断 JSON）

## Agent Skills

- [ ] **AC13：兼容 Skill 发现**；Windcode、Claude Code、Codex 和 Pi 显式配置目录中的有效 Agent Skill 均可发现。（验证：在四类根目录放置相同规范夹具并比较记录）
- [ ] **AC13：发现阶段只读元数据**，不读取 `SKILL.md` 正文、引用、脚本或资源。（验证：文件访问计数和不可读辅助文件探针）
- [ ] **AC13：初始上下文精简**，只包含 Skill 名称、短描述、来源和调用方式，不含正文及引用内容。（验证：检查首次模型请求）
- [ ] **AC13：按需加载正文**；匹配、`$name` 或声明式命令调用后才读取完整 Skill。（验证：比较调用前后读取记录和下一模型请求）
- [ ] **AC13、AC23：Skill 辅助文件有界**；引用仅在请求时读取，并拒绝越界符号链接、特殊文件和超大内容。（验证：正常与恶意引用夹具）
- [ ] **AC13：加载 Skill 不自动执行脚本**；脚本只有经现有工具显式调用并通过策略后才能运行。（验证：带哨兵脚本的 Skill 加载测试）
- [ ] **AC14：跨作用域同名 Skill 确定性覆盖**并向用户展示覆盖来源。（验证：用户级、项目级和运行级同名 Skill）
- [ ] **AC14：同一作用域重复 Skill 报错**，改变扫描顺序不会任意选择胜出项。（验证：随机化同层目录顺序并比较诊断）
- [ ] **AC24：Skill 运行缓存隔离**；当前运行加载后修改磁盘文件不会改变其内容，新快照和新运行才看到变化。（验证：跨运行修改正文并比较 digest）

## MCP 传输与能力

- [ ] **AC7：stdio MCP 正常生命周期**可以初始化、调用、取消和关闭，退出后子进程、文件句柄和后台任务归零。（验证：本地 stdio 契约 Server 与资源计数探针）
- [ ] **AC7：Streamable HTTP 正常生命周期**可以初始化、调用、取消和关闭，退出后连接与后台任务归零。（验证：本地 aiohttp 契约 Server 与连接计数探针）
- [ ] **AC7、AC22：连接边界**覆盖慢启动、连接超时、协议失败、Server 崩溃和运行中取消，且不遗留资源。（验证：逐类故障注入）
- [ ] **AC7：惰性连接**；仅发现或列出未激活扩展时不启动 MCP Server，首次需要能力时才连接。（验证：连接计数和进程探针）
- [ ] **AC7：恢复有界**；仅可恢复传输错误允许一次惰性重连，取消、认证和策略拒绝不重试。（验证：按错误类别检查连接次数）
- [ ] **AC8：四类 MCP 能力**的 Tools、Resources、Prompts 和 Instructions 均可发现并按需使用。（验证：能力齐全的本地 Server 契约测试）
- [ ] **AC8：MCP 来源边界**；进入目录、工具结果和模型上下文的内容都带 Server 来源并受大小限制。（验证：检查事件、ToolResult 和模型请求）
- [ ] **AC8：Resources 与 Prompts 受控访问**，只能通过运行时工具列出和获取，禁用或未信任 Server 不可访问。（验证：不同状态下调用管理工具）
- [ ] **AC8：Instructions 单独注入**，不能伪装为无来源核心 system 指令或覆盖 Windcode 安全规则。（验证：恶意 Instructions 提示注入夹具）
- [ ] **AC9：环境变量认证**可用于 stdio 或 HTTP，密钥值不进入普通配置、事件、轨迹、错误和模型上下文。（验证：使用唯一敏感标记扫描所有输出载体）
- [ ] **AC9：静态请求头引用**可完成 HTTP 认证，完整认证头和值均被脱敏。（验证：Server 校验请求头，同时扫描客户端记录）
- [ ] **AC9、AC18：明文凭据被拒绝**，插件与普通配置只能保存安全引用。（验证：在多个配置位置写入明文密钥并检查校验错误）
- [ ] **AC10：MCP Tool 复用统一参数校验**，复杂对象、数组、联合约束与附加属性规则保持原始 JSON Schema 语义。（验证：合法与非法复杂参数契约测试）
- [ ] **AC10：MCP Tool 复用调度与运行控制**，包括进度、预算、取消、结果截断、审计和结构化错误。（验证：完整调用链集成测试）
- [ ] **AC10、AC22：MCP 大输出外置**；完整内容进入本地 artifact，模型与终端只收到可解析摘要和引用。（验证：超大文本、图片及二进制结果）
- [ ] **AC11：未知 MCP Tool 默认审批**，即使 Server 标记为只读也不会自动放行。（验证：带只读注解的未知工具调用）
- [ ] **AC11：固定工具策略不突破上限**；配置只能收紧或缓存当前权限已允许的决定，不能绕过父权限、沙箱和网络限制。（验证：受限父运行中的放宽尝试）
- [ ] **AC12：小工具集直接暴露**，数量不超过阈值时完整 Schema 可供模型调用。（验证：阈值以下目录测试）
- [ ] **AC12：大工具集渐进暴露**，超过阈值时常驻上下文只有精简目录和搜索入口。（验证：阈值以上首次模型请求体积）
- [ ] **AC12：搜索激活在下一模型步生效**，只加入目标完整 Schema，不修改其他运行或全局快照。（验证：两个并发运行分别激活不同工具）

## Hooks

- [ ] **AC15：会话生命周期 Hooks**在 session start/end 的准确位置执行。（验证：记录一次新会话和关闭时间线）
- [ ] **AC15：用户与运行 Hooks**覆盖 user submit、run start/end/error，并保留 session/run/correlation ID。（验证：成功、失败和取消运行）
- [ ] **AC15：工具与权限 Hooks**覆盖 tool before policy、permission request 和 tool after，顺序与策略链一致。（验证：带审批工具调用的事件时间线）
- [ ] **AC15：压缩 Hooks**覆盖 compact before/after，载荷有界且不泄露被隐藏的完整历史。（验证：触发上下文压缩并检查载荷）
- [ ] **AC15：子智能体 Hooks**覆盖 subagent start/end，包含父子关联、角色和终态，不允许 Hook 借此启动新委派。（验证：成功与取消子任务）
- [ ] **AC16：预 Hook 可以拒绝**危险工具，拒绝发生在审批、`ToolStarted` 和实际副作用之前。（验证：带文件哨兵的拒绝测试）
- [ ] **AC16：预 Hook 只能收紧** effects、路径和网络限制，多个 Hook 的约束只增不减。（验证：先收紧后尝试放宽）
- [ ] **AC16：后置 Hook 不能篡改结果**，只能观察、通知或向下一模型步追加带来源上下文。（验证：尝试替换 ToolResult 和历史消息）
- [ ] **AC16：Hook command 复用安全链**，经过参数校验、权限、审批、沙箱、超时、输出限制、取消和审计。（验证：允许、拒绝、超时和取消命令）
- [ ] **AC16：Hook 递归保护**阻止同源 command 再次触发无限 Hook 链，首版不自动重试。（验证：自触发 Hook 并检查执行次数）
- [ ] **AC17：可选 Hook 故障隔离**，超时或异常不会破坏其他 Hook、扩展或核心运行。（验证：一个失败 Hook 与一个正常 Hook 并存）
- [ ] **AC17：安全 Hook fail closed**，判定型 Hook 异常不能把原本拒绝或未决的操作变为允许。（验证：拒绝链中注入异常）

## 故障隔离、事件与安全

- [ ] **AC17：可选 Skill、插件和 MCP 故障隔离**，其他扩展及核心 Agent 仍可使用。（验证：四类可选组件分别解析、启动和执行失败）
- [ ] **AC17：必需扩展失败明确终止**相关重载或运行，并显示失败阶段、来源、类别和修复建议。（验证：必需插件、Hook 和 MCP 分别失败）
- [ ] **AC20：扩展事件完整**覆盖发现、连接、激活、Skill、Hook、MCP、权限、失败、关闭和重载。（验证：完整插件运行后枚举事件种类）
- [ ] **AC20：事件可关联**，包含 snapshot generation、extension/source ID、session/run ID 及适用的 server/hook/call ID。（验证：检查完整审计时间线）
- [ ] **AC20：事件先持久化再发布**，消费者收到事件时对应会话记录已经存在。（验证：在 EventBus 消费回调中立即读取 SessionStore）
- [ ] **AC20：旧事件消费者兼容**，旧会话无需迁移且可以忽略未知扩展事件。（验证：读取旧会话夹具并使用只识别旧事件的消费者）
- [ ] **AC22：所有外部操作有超时和取消**，慢速扩展不能无限阻塞启动、重载、运行或退出。（验证：扫描、连接、调用、Hook 和关闭阶段分别注入延迟）
- [ ] **AC22：运行关闭树完整收敛**，正常完成、失败、取消和 SDK 关闭后无子进程、连接、文件句柄、异步任务或审批等待。（验证：四类路径的资源计数探针）
- [ ] **AC22：可选扩展资源耗尽不扩散**，超大输出、崩溃或错误协议不破坏其他扩展和会话持久化。（验证：并发运行正常与恶意扩展）
- [ ] **AC23：发现阶段零执行与零网络**，清单、Skill、Hook 和 MCP 元数据均作为不可信输入处理。（验证：恶意元数据与系统调用探针）
- [ ] **AC23：权限不可提升**，任何扩展配置组合都不能绕过审批、沙箱、工作区、网络和运行预算。（验证：组合插件尝试放宽全部安全边界）
- [ ] **AC23：诊断可操作且不泄密**，包含阶段、来源标签、类别和建议，不包含密钥、完整敏感路径或无限外部输出。（验证：对各失败阶段扫描诊断）
- [ ] **AC24：大型目录不线性膨胀上下文**，未激活 Skill 正文和 MCP Schema 不进入常驻模型请求。（验证：扩大能力数量并比较请求体增长只包含精简元数据）
