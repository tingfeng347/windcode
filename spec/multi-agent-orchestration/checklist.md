# Windcode 临时多智能体并行编排 Checklist

> 每一项都通过运行代码、执行测试或观察用户可见行为验证。先记录实际证据，再勾选通过。

## 委派与容量

- [x] **AC1：单个临时子任务**可以由主智能体创建，执行一个自包含任务后进入终态并退出。（验证：运行单任务模拟场景，观察创建、运行和终态事件）
- [x] **AC1：批量临时子任务**可以在一次请求中创建，子智能体不具备再次委派的能力。（验证：运行批量模拟场景，并让子模型尝试调用委派工具，期望工具不可见）
- [x] **AC2：显式委派模式**下，普通编码请求不会创建子智能体，明确要求并行或委派时可以创建。（验证：分别运行普通请求和明确委派请求，对比子任务事件）
- [x] **AC2：主动委派模式**启用后，主智能体可自行创建子任务，当前模式在状态信息中可见。（验证：加载主动模式配置并运行适合并行的模拟任务）
- [x] **AC3：默认并发上限**为 4；提交 6 个受屏障控制的任务时只观察到 4 个同时运行，另外 2 个排队。（验证：运行并发集成测试并记录峰值）
- [x] **AC3：FIFO 排队**在运行任务释放槽后按输入顺序启动等待任务。（验证：依次释放受控任务，观察启动序号）
- [x] **AC3：创建总量上限**为 8；第 9 个任务被结构化拒绝，且整批校验失败时不产生部分任务。（验证：运行容量边界测试）
- [x] **AC4：自包含任务校验**拒绝缺少目标、上下文、预期产物或验证要求的请求，并指出缺失字段。（验证：逐项省略字段运行参数测试）

## 上下文、角色与权限

- [x] **AC5：父会话隔离**生效；子模型请求不包含父历史消息或父工具结果。（验证：在父历史中放置唯一标记，检查子模型收到的请求中不存在该标记）
- [x] **AC5：兄弟任务隔离**生效；一个子任务的私有标记不会出现在另一个子任务上下文中。（验证：并行运行两个带不同标记的任务并检查请求）
- [x] **AC5：必要上下文继承**生效；子任务能看到自身任务、有效项目指令、工作区和模型配置。（验证：在临时仓库放置项目指令并观察子请求）
- [x] **AC6：权限上限**生效；子智能体无法获得高于父任务的权限、网络、沙箱或工作区范围。（验证：在受限父模式下尝试写入、联网和越界访问）
- [x] **AC6：工具最小化**生效；角色工具、任务工具和父工具取交集，任务不能通过参数扩大工具集。（验证：运行角色与工具交集测试）
- [x] **AC7：读取型并行任务**可以同时读取父工作区且不会互相阻塞。（验证：运行多个带同步屏障的读取任务）
- [x] **AC7：读取型写入拒绝**生效；文件写入、补丁、越界写入或产生修改的 Shell 操作均失败，父工作区保持不变。（验证：记录运行前后 Git 状态和文件摘要）

## Worktree 与集成

- [x] **AC8：干净 Git 基线**产生独立分支和 Worktree，两个写任务的路径、分支和修改互不相同。（验证：在临时仓库并行启动两个写任务并检查 Git Worktree 列表）
- [x] **AC8：父工作区隔离**生效；子提交集成前，父工作区文件和 HEAD 均不改变。（验证：对比创建前后父 HEAD、状态和文件摘要）
- [x] **AC8：脏工作区阻塞**生效；存在已跟踪、未跟踪、暂存或冲突修改时写任务被拒绝，读取任务仍可运行。（验证：分别构造四种 Git 状态）
- [x] **AC8：缺少 Git 或 Worktree 能力**时写任务明确降级失败，读取任务仍可运行。（验证：在非 Git 临时目录和模拟能力缺失环境运行）
- [x] **AC9：成功写任务交付**包含提交标识、变更摘要、验证命令和实际结果，且子 Worktree 干净。（验证：运行写任务集成测试并检查结构化结果）
- [x] **AC9：失败交付不冒充成功**；验证失败、提交失败或遗留未提交文件时状态和原因准确。（验证：分别构造三种失败）
- [x] **AC10：无冲突集成**在父智能体检查后通过 `cherry-pick` 完成，并在父工作区重新运行验证。（验证：执行集成场景，检查父提交和验证证据）
- [x] **AC10：集成冲突**会中止操作并恢复父 HEAD，不强制覆盖文件，同时保留子分支和 Worktree。（验证：构造同一行冲突并检查 Git 状态）
- [x] **AC10：父级验证失败**进入 `integration_failed`，保留已集成提交、失败证据和子 Worktree，不报告完全成功。（验证：让父级验证命令返回非零）

## 状态、审批与取消

- [x] **AC11：状态生命周期**可观察排队、运行、阻塞、完成、失败和取消；写任务还可观察集成、冲突或集成验证失败。（验证：运行覆盖各终态的模拟场景）
- [x] **AC11：事件与结果顺序**正确；实时事件按实际完成时间出现，批量最终结果按原始任务顺序返回。（验证：让后提交的任务先完成并比较两种顺序）
- [x] **AC12：来源审批**显示子任务身份、角色、工具、参数摘要和风险，响应只送达发起请求的子任务。（验证：并发触发两个审批并交叉响应）
- [x] **AC12：子任务澄清阻塞**生效；子智能体不能直接弹出用户问答，而是以 `blocked` 和原因返回父智能体。（验证：让模拟子模型调用用户问答工具）
- [x] **AC13：父任务取消传播**停止运行中模型请求、工具、审批等待和排队任务，取消后不再启动新工作。（验证：在四类状态下取消父任务）
- [x] **AC13：单任务取消隔离**生效；取消一个运行或排队任务时其他兄弟任务继续完成。（验证：取消指定任务并等待其余结果）

## 恢复、审计与接口

- [x] **AC14：成功清理**只删除已集成且干净的 Worktree 和临时分支。（验证：完成成功集成后检查 Git Worktree 列表和目录）
- [x] **AC14：保守保留**覆盖未集成、失败、冲突、集成验证失败和未提交变更，最终报告包含保留路径与原因。（验证：逐类构造遗留状态）
- [x] **AC14：异常恢复**把遗留排队或运行任务标记为中断后的取消，展示遗留成果，但不续跑模型、不集成、不重放副作用。（验证：在状态落盘后模拟进程退出并重开会话）
- [x] **AC15：本地审计时间线**包含父子关系、角色、状态、耗时、用量、工具、审批、提交、集成和清理结果。（验证：读取一次完整运行的父会话、子会话和轨迹）
- [x] **AC15：隐私与脱敏**保持现有配置语义，父级进度不泄漏被隐藏的工具参数，且运行期间没有新增遥测请求。（验证：使用敏感测试值检查持久化记录和网络模拟器）
- [x] **AC16：TUI 与 SDK 事件语义一致**，两者均可观察创建、排队、运行、审批、完成、失败、取消和用量。（验证：对同一模拟任务比较事件种类和关联字段）
- [x] **AC16：SDK 控制面**可查询稳定排序的子任务快照、取消指定任务并集成合格提交。（验证：运行 SDK 集成测试）
- [x] **AC17：并发状态隔离**保证会话、消息、工具状态、用量和结果不串线，一个子任务失败不破坏其他任务。（验证：运行带唯一标记和单点故障的并发场景）
- [x] **AC18：独立预算**在模型步骤、工具调用或运行时长耗尽时只终止对应子任务，并报告具体类别。（验证：分别设置极小子预算运行）
- [x] **AC18：聚合预算**耗尽后拒绝后续消费或任务启动，并报告父级累计用量。（验证：多个子任务共同耗尽聚合预算）
- [x] **AC19：单智能体兼容**保持原有编码任务、权限模式、旧会话读取、TUI 和 SDK 流程通过。（验证：运行原有单智能体集成与端到端测试）

## 端到端场景

- [x] **场景 1：并行读取**。用户明确要求并行分析两个独立模块，主智能体创建两个读取型子任务；两者并行完成，父智能体按原始顺序汇总，工作区无变化。（验证：运行无网络模拟模型，检查事件、结果和 Git 状态）
